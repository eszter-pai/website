Fine-Tuning LLM for Higher-Order Code Generation

October 2024

Overview

This project explores the challenges of teaching Large Language Models to generate higher-order code rather than simple first-order instructions. Developed as part of a university module at Universität Potsdam, the work investigates common LLM code generation issues through the lens of the COCOBOTS domain—a code generation task where models must instruct bots to place objects on game boards.

Motivation

Current LLMs face significant limitations when generating functions in the COCOBOTS domain. The need for accurate function generation is critical as it directly impacts developer productivity and code quality. Improved accuracy in function generation can:

  • Aid developers by automating repetitive coding tasks
  • Reduce errors through consistent pattern recognition and abstraction
  • Enhance overall productivity by generating maintainable, reusable code

The Problem: First-Order vs. Higher-Order Code

Many LLMs struggle with abstract code generation. When given instructions like "Stack a nut and washer in the 5th row and 3rd column using red for the nut and yellow for the washer," models often generate verbose, repetitive first-order code:

First-Order Code (Explicit Instructions):

put(board, "nut", "red", 4, 3)
put(board, "washer", "yellow", 4, 3)

While functional, this approach doesn't demonstrate true code abstraction. The ideal solution should use higher-order functions that encapsulate the logic and make the code more maintainable and reusable:

Higher-Order Code (Abstract Functions):

def m5(board, colors, x, y):
    shapes = ["nut", "washer"]
    for shape, color in zip(shapes, colors):
        put(board, shape, color, x, y)

m5(board, ["red", "yellow"], 4, 2)

Project Goals

The primary objective was to fine-tune smaller LLMs using LoRA (Low-Rank Adaptation) techniques to generate higher-order code consistently. This involved:

  • Evaluating multiple models (3B to 13B parameters) to find the optimal balance between efficiency and performance
  • Creating specialized training datasets with both first-order (4144 samples) and higher-order (1072 samples) COCOBOTS code
  • Implementing LoRA fine-tuning to adapt models efficiently without full retraining
  • Measuring performance using Exact Match, CodeBLEU, and Execution Success metrics
  • Analyzing the model's ability to abstract patterns and handle both code generation paradigms

The COCOBOTS Domain

COCOBOTS is a code generation benchmark where natural language instructions describe spatial arrangements of objects on a grid-based board. The task requires understanding:

  • Spatial reasoning: Interpreting positions like "5th row, 3rd column"
  • Object properties: Managing attributes such as colors, shapes, and types
  • Sequential operations: Stacking, placing, and arranging multiple objects
  • Code abstraction: Recognizing patterns that can be generalized into functions

Fine-Tuning Approach

Using LoRA (Low-Rank Adaptation), we fine-tuned a base language model on curated examples of higher-order COCOBOTS code. LoRA enables efficient adaptation by injecting trainable low-rank matrices into the transformer layers, significantly reducing the computational cost compared to full fine-tuning.

The training process focused on teaching the model to recognize patterns in object placement instructions and translate them into reusable function definitions rather than repetitive explicit commands.

Key Findings

  • Pretrained Model Limitations: All pretrained models showed 0% performance on both first-order and higher-order tasks, indicating they require fine-tuning for COCOBOTS-specific code generation
  • Fine-Tuning Impact: After fine-tuning on first-order data, models achieved significant improvements in first-order tasks (ranging from 9.8% to 100% Exact Match), but struggled with higher-order generalization
  • Execution Success Advantage: Models demonstrated high execution success rates (81-100%) on first-order tasks after fine-tuning, showing that generated code is often functionally correct even without exact matches
  • Mistral's Outstanding Performance: Mistral-7B achieved perfect scores (100%) across all first-order metrics after fine-tuning, demonstrating exceptional learning from the training data
  • Cross-Order Generalization Gap: Models trained on first-order code showed minimal ability to generate higher-order code, with most achieving 0% Exact Match on higher-order tasks, highlighting the abstraction challenge

Common LLM Code Generation Issues

This project highlighted several recurring problems in LLM-based code generation:

  • Verbosity: Models tend to produce unnecessarily explicit code rather than leveraging abstraction
  • Lack of DRY Principle: Repeated code blocks instead of function definitions
  • Context Limitations: Difficulty maintaining consistency across longer code sequences
  • Domain-Specific Logic: Struggles with understanding specialized domains without targeted training

Comparative Analysis

Comparing the two training approaches reveals important insights about code abstraction learning:

  • Higher-Order Performance: Models trained on higher-order code achieved significantly better higher-order generation (up to 26.15% vs 0% Exact Match for Llama-3.1), demonstrating the importance of exposure to abstracted patterns
  • Bidirectional Capability: Higher-order training maintained reasonable first-order performance, suggesting models can learn to generate both abstraction levels when trained on complex examples
  • CodeBLEU Improvements: Higher-order training showed enhanced CodeBLEU scores for higher-order tasks (72.76% to 81.66%), indicating better structural similarity to reference code
  • Execution Success Advantage: Higher-order trained models achieved strong execution success on higher-order tasks (up to 83.80%), showing functional correctness despite lower exact matches
  • Trade-offs: While first-order training excelled at first-order tasks (100% for Mistral), higher-order training provided more balanced performance across both abstraction levels

Combined Training Analysis

The combined training approach reveals the benefits and limitations of exposing models to diverse code patterns:

  • Best First-Order Performance: Combined training achieved exceptional first-order results (95.40% to 100% Exact Match), matching or exceeding first-order-only training performance
  • Improved Higher-Order Scores: Models showed significant gains in higher-order generation (up to 44.62% for CodeLlama-7b) compared to first-order-only training, though not as strong as higher-order-only training
  • Balanced CodeBLEU Metrics: CodeBLEU scores improved across both tasks, with CodeLlama-7b achieving 98.36% first-order and 90.57% higher-order, demonstrating better structural code quality
  • Optimal Execution Success: Combined training produced the most reliable executable code, with three models achieving 100% execution success on first-order tasks and improved higher-order execution rates (up to 69.20%)
  • Training Strategy Recommendation: For production systems requiring versatility, combined training offers the best compromise—excellent first-order performance with reasonable higher-order capabilities, making it the most practical approach for real-world applications

Practical Applications

While this project focused on the COCOBOTS domain, the findings have broader implications:

  • Improving code generation tools for software developers
  • Teaching LLMs to write cleaner, more maintainable code
  • Developing better benchmarks for evaluating code abstraction capabilities
  • Informing prompt engineering strategies for code-generation tasks

Lessons Learned

This university project provided valuable insights into the limitations and capabilities of LLMs for code generation. The gap between first-order and higher-order code generation represents a significant challenge in making AI-assisted programming tools more effective. Fine-tuning with techniques like LoRA offers a promising path forward, but achieving consistent abstraction remains an open research problem.

Future Directions

  • Exploring reinforcement learning approaches to reward code abstraction
  • Developing better evaluation metrics for code quality beyond functional correctness
  • Investigating multi-stage generation pipelines that explicitly separate abstraction from implementation
  • Extending the approach to more complex programming paradigms and languages
